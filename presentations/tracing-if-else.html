Tracing if / else

Working out what Python will do.

Not "what does this code do?" but "which line actually runs, and why?"

1

What is tracing?

Tracing means reading code line by line and working out what each part does, without running it.

You are playing the role of the computer.

The goal is to work out the output before Python does.

2

The four steps

Every trace follows these steps.

  1. Find the variables. Note their starting values.
  2. Reach the if. Read the condition.
  3. Substitute and evaluate. Replace the variable with its value. Is the condition True or False?
  4. Follow the correct path. True runs the if block. False runs the else block.
►cups = 8
if cups > 5:
print("Great selling!")
else:
print("Keep going!")

Step 1 of 4 · Find the variables

What values do the variables hold?

Variablecups = 8

Every time we see cups in the code, we can replace it with 8.

cups = 8
►if cups > 5:
print("Great selling!")
else:
print("Keep going!")

Steps 2 and 3 · Substitute and evaluate

What is the condition?

Python reaches the if line. It needs to evaluate the condition before it can decide what to do.

cups > 5  →  8 > 5  →  True

cups holds 8, so the condition becomes 8 > 5. That is True.

cups = 8
if cups > 5:
► print("Great selling!")
else:
print("Keep going!")

Step 4 · Follow the path

Condition was True.

The if block runs.

The else block is skipped entirely. Python never reads it.

Output
Great selling!

3

Notice

Same code. Different output.

Only the value of cups changed.

True runs the if block. False runs the else block.

The path through the code depends entirely on the values when the condition is checked. That is the whole point of if / else.
