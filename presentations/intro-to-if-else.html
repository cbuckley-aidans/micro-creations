Introduction
0

New concept

if / else

How programs make decisions.

So far our code has run every line in order. Conditionals let it choose between different paths.

1

The idea

You make decisions all day.

IF it is raining → take an umbrella.
ELSE → wear sunglasses.

Programs make decisions in exactly this way.

2

The condition

Every decision starts with a question.

A condition is a question that is either true or false.

Is it raining?  →  true or false

There is no maybe. The program needs a clear yes or no.

3

Two paths

if … else

if the answer is true, do the first thing.

else, which means otherwise, do the second thing.

One question. Two paths. The program always takes exactly one.

4

Drawing a decision

The diamond asks the question. The boxes are what we do.

5

Modelled example

In a game, if your score is 10 or more, the player wins. If not, the game says try again.

Let's turn this into a flowchart, then into code.

6

Step 1 · Find the condition

What is the yes/no question?

Is the score 10 or more?

In code we write that as:

score >= 10

>= means "greater than or equal to".

7

Step 2 · Draw the flowchart

Same shape as the umbrella. One question, two outcomes.

8

Step 3 · Write the code

if score >= 10: print("You win!") else: print("Try again")

The diamond's question becomes the if line. It ends with a colon.

The YES box sits indented under the if.

The NO box sits indented under else.

9

The shape of if / else

if a question is true: do this else: do this instead

Question, colon, indent. Then else, colon, indent.

Every if/else you write has this same shape.
